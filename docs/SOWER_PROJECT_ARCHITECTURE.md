# Архитектура проекта сеялки автовысева на MKS TinyBee V1.0

**Дата:** 2026-05-15  
**Плата:** MakerBase MKS TinyBee V1.0  
**Назначение:** автономная машина высева семян в кассеты, подаваемые конвейерной лентой  
**База прошивки:** grblHAL/ESP32 как motion + I2S layer, поверх него прикладная FSM сеялки  
**Правила кода:** `SOWER_CODE_ARCHITECTURE_RULES.md`

---

## 1. Концепция машины

Кассета подается конвейерной лентой. Оптический датчик определяет передний край кассеты, после чего прошивка индексирует кассету на центр первого ряда ячеек. Головка стационарная. Инструменты работают над текущим рядом:

- система лунок делает углубления под семена;
- ряд из 8 вакуумных игл захватывает 8 семян;
- механизм переноса поворачивает иглы по полуокружности к зоне сброса;
- семена сбрасываются в ячейки;
- иглы продуваются отдельным трактом давления;
- конвейер индексирует кассету на следующий ряд.

Первый вариант машины рассчитан на один тип кассеты, но конфигурация должна хранить геометрию кассеты и параметры процесса так, чтобы потом можно было добавить рецепты.

### Зафиксированные решения v1

- Привод конвейера: шаговый двигатель Leadshine `42CM08`, передача от мотора к валу ленты ремнем `1:8`.
- Драйверы шаговых моторов: `TMC2209`, рабочий ток на старте проекта `1.5 A`.
- Кассета двигается только конвейером; отдельного подвижного X/Y портала нет.
- Home/нулевая метка конвейера не нужна: лента запускается и подает кассету до срабатывания оптического датчика начала кассеты.
- Система лунок делает сразу 8 лунок на ряд.
- Вакуум общий на ряд из 8 игл; контроль захвата идет по одному датчику вакуума линии.
- Дисплей: ILI9341 без touch, управление через энкодер.
- Органы управления: кнопка Start/Stop и аппаратная кнопка E-stop.

## 2. Выбранная аппаратная раскладка TinyBee

TinyBee имеет 5 независимых STEP/DIR/ENABLE каналов через I2S-расширитель. Для этой механики каналы используются так:

| Функция | Канал TinyBee | Назначение |
|---------|---------------|------------|
| Конвейер | X, `I2SO(0..2)` | Leadshine `42CM08`, привод вала ленты через ремень `1:8` |
| Резерв / второй привод конвейера | Y, `I2SO(3..5)` | Не использовать в v1, оставить под будущую механику |
| Система лунок | Z, `I2SO(6..8)` | Отдельный шаговый инструмент лунок |
| Перенос игл | E0, `I2SO(9..11)` | Полукруг захвата/сброса |
| Резерв | E1, `I2SO(12..14)` | Будущий узел или второй инструмент |

В v1 конвейер считать одной осью: мотор вращает вал ленты через ременную передачу `1:8`. Если позже понадобится второй мотор конвейера, его нельзя просто добавить без проверки синхронизации: либо механическая связь валом/ремнем, либо firmware slaving на отдельном драйвере.

Конвейер не делает homing. В состоянии `WAIT_CASSETTE` прошивка запускает ленту с подающей скоростью и останавливает ее по оптическому датчику кассеты, затем индексирует кассету на `first_row_offset_from_sensor_mm`.

### Входы

| Функция | GPIO | Примечание |
|---------|------|------------|
| Оптический датчик начала кассеты | GPIO35 `MT_DET` | Цифровой вход, только input |
| Home системы лунок | GPIO33 `X_STOP` | Концевик/оптодатчик инструмента |
| Home переноса игл | GPIO32 `Y_STOP` | Концевик/оптодатчик полуокружности |
| Кнопка Start/Stop | GPIO22 `Z_STOP` | Дискретный вход панели оператора |
| Датчик вакуума | GPIO36 `TH1 ADC` | Аналоговый вход |
| Резерв ADC | GPIO39 `TB ADC` | Давление/вакуум/диагностика |
| Резерв ADC или SD detect | GPIO34 `TH2/SD_DET` | Использовать только после выбора джампера |
| Энкодер A | GPIO14 `BTN_EN1` | EXP2; проверить подтяжки |
| Энкодер B | GPIO12 `BTN_EN2` | EXP2; strapping pin, не тянуть вверх на boot |
| Энкодер кнопка | GPIO13 `BTN_ENC` | EXP1 |

### Выходы

| Функция | Канал TinyBee | Примечание |
|---------|---------------|------------|
| Насос вакуума или внешний relay enable | `H-BED`, `I2SO(16)` | Через реле/SSR, с учетом допустимого тока |
| Клапан вакуума | `H-E0`, `I2SO(17)` | Модель неизвестна; нужна проверка напряжения/тока и flyback-защита |
| Клапан давления/продувки | `H-E1`, `I2SO(18)` | Модель неизвестна; взаимоисключение с вакуумом в software |
| DC мотор вибробункера | `FAN1`, `I2SO(19)` | Модель неизвестна; PWM только через подходящий силовой драйвер |
| Резерв клапана/сброса | `FAN2`, `I2SO(20)` | PWM или on/off |
| Зуммер/индикация | `BEEPER`, `I2SO(21)` | Диагностика |

### Дисплей ILI9341

ILI9341 подключать к SPI платы:

| Сигнал | GPIO | Примечание |
|--------|------|------------|
| SCK | GPIO18 | EXP2 / SPI |
| MOSI | GPIO23 | EXP2 / SPI |
| MISO | — | Не использовать в v1: touch не нужен |
| CS | GPIO5 | SPI CS |
| DC | GPIO21 | EXP1, свободен без LCD12864 |
| RST | GPIO4 | EXP1, свободен без LCD12864 |
| Backlight | Питание или GPIO2 | Предпочтительно постоянно включить через питание; GPIO2 только если нужен PWM |

В v1 не использовать SD-карту одновременно с дисплеем без явного SPI bus arbitration. GPIO0 не использовать для кнопок и подтяжек, влияющих на boot. GPIO12 использовать для энкодера только со схемой, которая не ломает strapping при включении ESP32.

### E-stop

E-stop должен аппаратно разрывать питание силовой части 24 V или ENABLE драйверов/реле. GPIO-вход для статуса E-stop можно добавить позже, но он не заменяет аппаратную цепь безопасности.

## 3. Архитектура прошивки

Проект делится на слои:

```text
app_sower
  sower_fsm          - главный цикл высева
  recipe             - геометрия кассеты и параметры процесса
  row_process        - операции одного ряда: лунки, захват, перенос, сброс
  fault_manager      - ошибки, retry, критичность, журнал

machine
  cassette_indexer   - датчик кассеты, смещение к первому ряду, шаги между рядами
  dibbler            - система лунок, homing, рабочий ход
  transfer_arm       - полуокружность игл, homing, позиции PICK/DROP/BLOW
  vacuum_system      - вакуум, датчик, пороги, retry захвата
  pneumatics         - клапаны вакуума/давления, взаимные блокировки
  vibro_feeder       - DC PWM вибробункера

platform
  motion_adapter     - grblHAL calls / синхронные движения осей
  board_tinybee      - I2S bits, GPIO, ADC, PWM, display pins
  ui_display         - ILI9341 экран, статусы, меню
  ui_web             - Wi-Fi конфиг, рецепты, сервисные команды
  storage            - NVS/LittleFS: конфиг, рецепты, журнал
  safety             - E-stop status, watchdog, safe outputs
```

grblHAL использовать как источник готового motion planner, step generation и I2S output для TinyBee. Прикладная логика не должна дергать STEP/DIR вручную. Она вызывает абстрактные команды типа `move_conveyor_mm()`, `home_transfer()`, `move_dibbler_to_work()`.

### CPU budget

ESP32-WROOM-32U достаточно для v1, потому что realtime motion вынесен в grblHAL/I2S, а технологическая FSM, ADC, клапаны, энкодер и веб-конфиг имеют низкую вычислительную нагрузку. Главные ограничения:

- ILI9341 обновлять через dirty rectangles, без full-screen animation во время производственного цикла;
- Wi-Fi/web не должен блокировать `control_task` и motion;
- запись flash делать асинхронно через log/storage task;
- при bring-up измерять CPU idle/headroom, stack high watermark и задержку реакции на датчик кассеты;
- если CPU headroom меньше 30%, сначала снижать refresh дисплея и ограничивать web во время production cycle.

## 4. Основной цикл FSM

```text
BOOT
  -> SAFE_OUTPUTS_OFF
  -> LOAD_CONFIG
  -> INIT_BOARD
  -> HOME_TOOLS
  -> IDLE

IDLE
  -> WAIT_CASSETTE on Start

WAIT_CASSETTE
  -> RUN_CONVEYOR_FEED
  -> DETECT_CASSETTE_EDGE when optical sensor active
  -> FAULT if timeout enabled

DETECT_CASSETTE_EDGE
  -> STOP_CONVEYOR
  -> INDEX_TO_FIRST_ROW_CENTER

INDEX_TO_FIRST_ROW_CENTER
  -> PROCESS_ROW(row = 0)

PROCESS_ROW
  -> MAKE_HOLES
  -> PICK_SEEDS
  -> VERIFY_PICKUP
  -> TRANSFER_TO_DROP
  -> DROP_SEEDS
  -> BLOW_NEEDLES
  -> LOG_ROW_RESULT
  -> ADVANCE_TO_NEXT_ROW or EJECT_CASSETTE

ADVANCE_TO_NEXT_ROW
  -> PROCESS_ROW(row + 1)

EJECT_CASSETTE
  -> IDLE or WAIT_CASSETTE

FAULT
  -> RETRY, SKIP_ROW, PAUSE, or ABORT depending on fault policy
```

### Операции ряда

| Операция | Поведение |
|----------|-----------|
| `MAKE_HOLES` | Перевести инструмент лунок в рабочий ход, выдержать задержку, вернуть в безопасное положение |
| `PICK_SEEDS` | Включить вакуум, включить вибробункер, перевести иглы в позицию захвата |
| `VERIFY_PICKUP` | Ждать достижения порога вакуума; при провале выполнить retry |
| `TRANSFER_TO_DROP` | Остановить вибро, перенести иглы полуокружностью в позицию сброса |
| `DROP_SEEDS` | Выключить вакуум, открыть давление/сброс на заданное время |
| `BLOW_NEEDLES` | Импульс продувки, затем закрыть все клапаны |
| `ADVANCE_TO_NEXT_ROW` | Сдвинуть конвейер на `row_pitch_mm` |

## 5. Конфигурация и рецепты

Для v1 хранить один активный рецепт, но структура должна поддерживать список рецептов позже.

Минимальные параметры:

```text
recipe:
  name
  rows
  needles = 8
  row_pitch_mm
  first_row_offset_from_sensor_mm
  conveyor_motor_model = Leadshine 42CM08
  stepper_driver = TMC2209
  stepper_run_current_a = 1.5
  conveyor_belt_reduction = 8.0
  conveyor_steps_per_mm
  conveyor_sync_mode = single_motor_to_drive_shaft

dibbler:
  home_dir
  home_speed
  work_position_steps
  safe_position_steps
  dwell_ms

transfer_arm:
  home_dir
  home_speed
  pick_position_steps
  drop_position_steps
  blow_position_steps
  move_speed

vacuum:
  channels = 1
  needles_per_channel = 8
  pickup_threshold_adc
  pickup_timeout_ms
  hold_min_adc
  release_threshold_adc
  retry_count
  retry_blow_ms

pneumatics:
  vacuum_valve_on_delay_ms
  pressure_pulse_ms
  valve_deadtime_ms

vibro:
  pwm_duty_pick
  pre_pick_ms
  post_pick_ms

fault_policy:
  no_cassette
  pickup_failed
  vacuum_lost
  home_failed
  estop
```

Конфиг хранить в NVS. Для более крупных журналов и веб-файлов использовать LittleFS/SPIFFS во flash.

## 6. Ошибки и политика retry

| Ошибка | Критичность v1 | Действие |
|--------|----------------|----------|
| E-stop | Критическая | Немедленно safe outputs off, движение остановить |
| Не найден home инструмента | Критическая | Stop, требуется сервис |
| Нет кассеты после старта | Некритичная | Вернуться в WAIT_CASSETTE или timeout fault |
| Датчик кассеты сработал некорректно | Критичная для текущей кассеты | Stop/Pause, не продолжать индексирование |
| Не набран вакуум на захвате | Некритичная до лимита | Retry; после лимита skip row или pause по настройке |
| Потеря вакуума при переносе | Средняя | Retry row или mark failed |
| Не упал вакуум после сброса | Средняя | Blow retry, затем pause |
| ADC вне диапазона | Критичная | Stop, ошибка датчика |

Начальная политика: для `pickup_failed` делать 2-3 retry, логировать ряд, после исчерпания retry ставить паузу оператора. Для мини-производства позже можно добавить режим `skip_row_and_continue`.

## 7. UI и режимы работы

### Экран ILI9341

Экран v1 показывает:

- состояние машины: Idle, Homing, Waiting cassette, Row N, Fault;
- текущий рецепт и счетчик кассет/рядов;
- вакуум ADC и пороги;
- ручные сервисные команды: home, jog conveyor, test valves, test vibro;
- активную ошибку и действие оператора.

### Web UI

Web UI нужен для настройки:

- геометрия кассеты;
- steps/mm и скорости;
- пороги вакуума;
- длительности вакуума/давления/вибро;
- политика retry;
- экспорт журнала.

### Сервисные режимы

| Режим | Назначение |
|-------|------------|
| Manual jog | Проверка моторов и направления |
| Valve test | Проверка вакуума, давления, продувки |
| Vacuum calibration | Считать ADC без семян, с захватом, на утечке |
| Cassette calibration | Подобрать offset от датчика до первого ряда |
| Dry run | Цикл без вакуума и семян |

## 8. Реализация по этапам

### Этап 1: bring-up платы

- Собрать grblHAL/ESP32 под `MKS_TINYBEE_V1`.
- Проверить I2S STEP/DIR/ENABLE на всех осях.
- Настроить TMC2209 на рабочий ток `1.5 A` и проверить нагрев драйвера/мотора под нагрузкой.
- Проверить endstop GPIO, ADC вакуума, HE/FAN outputs.
- Проверить кнопки Start/Stop и энкодер.
- Проверить безопасное состояние всех выходов после boot/reset.

### Этап 2: machine layer

- Реализовать `board_tinybee` и `motion_adapter`.
- Завести оси: conveyor, dibbler, transfer_arm.
- Реализовать homing лунок и переноса.
- Реализовать ручные команды через serial/web.

### Этап 3: технологический цикл

- Реализовать FSM без UI, запуск через serial/web.
- Добавить конфиг рецепта в NVS.
- Добавить retry захвата по вакууму.
- Добавить журнал событий.

### Этап 4: UI

- Подключить ILI9341.
- Добавить экран статуса, сервисные тесты и настройки.
- Добавить web-конфиг и сохранение параметров.

### Этап 5: производственный режим

- Калибровка на реальной кассете.
- Настройка скоростей, задержек и порогов.
- Добавить статистику: кассеты, строки, retry, пропуски, аварии.

## 9. Открытые вопросы перед схемой подключения

1. Точные параметры Leadshine `42CM08`: шаг/оборот, допустимое напряжение, момент и нагрев при `1.5 A`.
2. Какая кинематика системы лунок: ход вверх/вниз, глубина, датчик home, нужен ли датчик нижнего положения?
3. Какие клапаны будут использоваться: напряжение, ток, нормально открытые или нормально закрытые.
4. DC мотор вибробункера питается от 12/24 V и какой у него ток.
5. Нужен ли GPIO-статус E-stop в прошивке или достаточно аппаратного разрыва силовой цепи.
6. Нужно ли оставлять GPIO19 свободным под будущий touch/SD/MISO или можно занять его под дополнительный вход.
